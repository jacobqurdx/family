# Stub Responses

Pre-written LLM responses used when `AGENT_STUB_LLM=true` or `--stub` is passed.

The stub backend matches responses by keyword inspection of the incoming prompt:
- `relevance/` ‚Äî `relevant_true.json` (tariff/WuXi/shortage keywords) or `relevant_false.json`
- `novelty/` ‚Äî `novel_true.json` or `novel_false.json`
- `severity/` ‚Äî matched to tier by keywords (`100%`/`embargo`‚Üícritical, `BIOSECURE`/`55%`‚Üíhigh_cdmo or high_tariff, etc.)
- `impact/` ‚Äî `impact_tariff.json` or `impact_cdmo.json`
- `briefing/` ‚Äî `investigation_report_template.md`
- `collection/` ‚Äî `web_search_results.json` (returned by client.search())

These files are generated by `agent.llm.write_stub_files()`. Safe to delete and re-run.
Do not edit these files ‚Äî they exist solely for deterministic offline testing.
