## Grading Criteria

Grade as CERTAIN if ALL of the following hold:
- The signal provides unambiguous, factual evidence directly supporting the assessment
- The assessed tier/value aligns clearly with the criteria for that tier (not a borderline case)
- There are no material contradictions between the signal content and the assessment reasoning
- The sensitivity context data cited (if any) exists and supports the claimed magnitude

Grade as UNCERTAIN if ANY of the following hold:
- The signal uses hedged language ("reportedly", "sources say", "may", "could") without confirming facts
- The assessed tier is borderline (e.g., could reasonably be HIGH or CRITICAL, or ELEVATED or HIGH)
- Key evidence cited in the reasoning is absent or ambiguous in the signal text
- The cost/timeline figures cited do not directly trace to the sensitivity data provided
- Multiple risk vectors are claimed simultaneously without clear evidence for each

## Instructions

Return ONLY a valid JSON object:

{{
  "grade": "CERTAIN" or "UNCERTAIN",
  "confidence": 0.0 to 1.0 (your confidence in the grade itself),
  "uncertainty_flags": ["flag 1", "flag 2"],
  "reasoning": "Two to three sentences explaining why you assigned this grade, citing specific text from the signal or assessment."
}}

uncertainty_flags should be an empty list [] if grade is CERTAIN.
uncertainty_flags should name the specific ambiguity (e.g., "hedged language: 'reportedly'", "borderline tier: signal describes proposal not enactment", "cost figure not in sensitivity data").
